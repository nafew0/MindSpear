# MindSpear - Local Development Setup Guide (WSL)

## Overview

MindSpear is an interactive educational platform combining **Quizzes** (Kahoot-style), **Quests** (Mentimeter-style), and **Surveys** in a single platform.

| Component | Stack               | Port  |
|-----------|----------------------|-------|
| Frontend  | Next.js 15 (React 19) | 2000  |
| Backend   | Laravel 12 (PHP 8.4)  | 8000  |
| Reverb    | Laravel Reverb         | 8080  |

---

## Prerequisites Checklist

Your WSL currently has:
- PHP 8.3.6 (needs **8.4+**)
- No Node.js inside WSL (only Windows Node via `/mnt/c/`)
- No Composer inside WSL
- Missing PHP extensions: `pdo_sqlite`, `pdo_mysql`, `gd`, `sqlite3`

---

## Step 1: Install Required System Packages

Run these commands inside your WSL terminal:

### 1.1 Add PHP 8.4 Repository

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y software-properties-common
sudo add-apt-repository ppa:ondrej/php -y
sudo apt update
```

### 1.2 Install PHP 8.4 + Required Extensions

```bash
sudo apt install -y php8.4 php8.4-cli php8.4-fpm php8.4-common \
  php8.4-mysql php8.4-sqlite3 php8.4-pgsql \
  php8.4-mbstring php8.4-xml php8.4-curl php8.4-zip \
  php8.4-gd php8.4-bcmath php8.4-intl php8.4-readline \
  php8.4-tokenizer php8.4-fileinfo php8.4-dom \
  php8.4-redis
```

### 1.3 Set PHP 8.4 as Default

```bash
sudo update-alternatives --set php /usr/bin/php8.4
php -v
# Should show PHP 8.4.x
```

### 1.4 Install Node.js (v20 LTS) Inside WSL

> Do NOT rely on the Windows Node.js at `/mnt/c/`. Install natively inside WSL.

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v   # Should show v20.x
npm -v    # Should show 10.x
```

### 1.5 Install Yarn (optional, npm works too)

```bash
sudo npm install -g yarn
```

### 1.6 Install Composer Inside WSL

```bash
cd ~
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer
composer -V
```

### 1.7 Install SQLite (for the default database)

```bash
sudo apt install -y sqlite3
```

## Step 2: Set Up the Backend (Laravel)

```bash
cd /home/bdren/Projects/MindSpear/backend
```

### 2.1 Copy the Environment File

The `.env` file has already been created for you (see below). If not:

```bash
cp .env.example .env
```

### 2.2 Install PHP Dependencies

```bash
composer install
```

### 2.3 Generate Application Key

```bash
php artisan key:generate
```

### 2.4 Create the SQLite Database

```bash
touch database/database.sqlite
```

### 2.5 Run Migrations

```bash
php artisan migrate
```

### 2.6 Seed the Super Admin (optional but recommended)

```bash
php artisan db:seed --class=SuperAdminSeeder
```

> Make sure you set `SUPER_ADMIN_EMAIL` and `SUPER_ADMIN_PASSWORD` in your `.env` first.

### 2.7 Build Frontend Assets (for Filament Admin Panel)

```bash
npm install
npm run build
```

### 2.8 Start the Backend Server

```bash
php artisan serve
# Runs on http://127.0.0.1:8000
```

Or use the combined dev command (server + queue + vite):

```bash
composer run dev
```

## Environment Files - What to Fill In

### Backend `.env` - Field Guide

| Variable                  | Required? | What to put                                                    |
|---------------------------|-----------|----------------------------------------------------------------|
| `APP_KEY`                 | **Auto**  | Generated by `php artisan key:generate`. Do NOT fill manually. |
| `APP_URL`                 | Yes       | `http://localhost:8000` for local dev                          |
| `DB_CONNECTION`           | Yes       | `sqlite` (default) or `mysql` if you prefer MySQL             |
| `DB_HOST`                 | If MySQL  | `127.0.0.1`                                                   |
| `DB_PORT`                 | If MySQL  | `3306`                                                        |
| `DB_DATABASE`             | If MySQL  | Your database name (e.g., `mindspear`)                        |
| `DB_USERNAME`             | If MySQL  | Your MySQL username                                           |
| `DB_PASSWORD`             | If MySQL  | Your MySQL password                                           |
| `MAIL_MAILER`             | Optional  | `log` for dev (emails go to `storage/logs/`). Use `smtp` for real emails. |
| `MAIL_HOST`               | If SMTP   | e.g., `smtp.gmail.com` or `smtp.mailtrap.io`                  |
| `MAIL_PORT`               | If SMTP   | `587` (TLS) or `465` (SSL)                                    |
| `MAIL_USERNAME`           | If SMTP   | Your SMTP username                                             |
| `MAIL_PASSWORD`           | If SMTP   | Your SMTP password / app password                              |
| `MAIL_FROM_ADDRESS`       | If SMTP   | e.g., `noreply@mindspear.app`                                 |
| `GOOGLE_CLIENT_ID`        | Optional  | For Google OAuth login. Get from Google Cloud Console.         |
| `GOOGLE_CLIENT_SECRET`    | Optional  | Paired with the Client ID above.                               |
| `GOOGLE_REDIRECT_URI`     | Optional  | `http://localhost:2000/auth/google/callback` for local dev     |
| `MICROSOFT_CLIENT_ID`     | Optional  | For Microsoft OAuth login. Get from Azure Portal.              |
| `MICROSOFT_CLIENT_SECRET` | Optional  | Paired with the Client ID above.                               |
| `MICROSOFT_REDIRECT_URI`  | Optional  | `http://localhost:2000/auth/microsoft/callback` for local dev  |
| `SUPER_ADMIN_EMAIL`       | Optional  | Email for the seeded admin user (e.g., `admin@mindspear.app`)  |
| `SUPER_ADMIN_PASSWORD`    | Optional  | Password for the seeded admin user                             |

### Frontend `.env` - Field Guide

| Variable                      | Required? | What to put                                                    |
|-------------------------------|-----------|----------------------------------------------------------------|
| `NEXT_PUBLIC_API_BASE_URL`    | **Yes**   | `http://localhost:8000/api/v1` - points frontend to your backend API |
| `NEXT_PUBLIC_REVERB_APP_KEY`  | **Yes**   | The Reverb app key from your backend `.env`                    |
| `NEXT_PUBLIC_REVERB_HOST`     | **Yes**   | `localhost` for local Reverb, or your websocket host in production |
| `NEXT_PUBLIC_REVERB_PORT`     | **Yes**   | `8080` locally unless you changed `REVERB_PORT`                |
| `NEXT_PUBLIC_REVERB_SCHEME`   | **Yes**   | `http` locally, `https` in production                          |
| `APP_URL`                     | Yes       | `http://localhost:2000` for local dev                          |
| `OPENAI_API_KEY`              | Optional  | Only if AI features are used. Get from OpenAI dashboard.       |

## Using MySQL Instead of SQLite (Optional)

If you prefer MySQL:

### Install MySQL in WSL

```bash
sudo apt install -y mysql-server
sudo service mysql start
sudo mysql_secure_installation
```

### Create the Database

```bash
sudo mysql -u root -p
```

```sql
CREATE DATABASE mindspear CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'mindspear'@'localhost' IDENTIFIED BY 'your_password_here';
GRANT ALL PRIVILEGES ON mindspear.* TO 'mindspear'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### Update Backend `.env`

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=mindspear
DB_USERNAME=mindspear
DB_PASSWORD=your_password_here
```

Then re-run migrations:

```bash
php artisan migrate:fresh
php artisan db:seed --class=SuperAdminSeeder
```

## Common Issues & Fixes

### "php: command not found" or wrong PHP version
```bash
sudo update-alternatives --set php /usr/bin/php8.4
```

### Canvas/native module build failures
```bash
sudo apt install -y build-essential libcairo2-dev libjpeg-dev libpango1.0-dev libgif-dev librsvg2-dev
```

### "SQLSTATE[HY000]: General error: 1 no such table"
Migrations haven't run. Run:
```bash
php artisan migrate
```

### Port already in use
```bash
# Find what's using the port
lsof -i :8000
lsof -i :2000
# Kill it
kill -9 <PID>
```

### Permission issues on storage/
```bash
cd /home/bdren/Projects/MindSpear/backend
chmod -R 775 storage bootstrap/cache
```
